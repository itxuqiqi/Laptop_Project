# 惠州石化成本报表自动填充项目

这是一个用于辅助生成惠州石化成本报表的本地自动填充项目。项目目标是基于成本报表模板、产销存收入表以及各类月度源数据，稳定生成完整成本报表，并减少新增月份数据取不到数的问题。

## 当前状态

- 已建立本地 Git 备份并推送到 GitHub 仓库：`itxuqiqi/Laptop_Project`
- 已建立独立 Obsidian 项目资料库：`惠州石化项目-Obsidian`
- 已修复一处“其他在途”读取兼容问题：暂估表页签名存在首尾空格时，代码会按去空格后的页签名继续匹配，避免整张表被跳过。

## 目录结构

```text
.
├── code/                         # 自动填充系统代码
├── 惠州石化项目-Obsidian/          # 项目资料库、日志、TODO、决策记录
├── .gitignore                    # 排除月度数据、Excel 文件、PDF、临时文件
└── README.md
```

月度 Excel 源数据、成本报表模板、PDF 和 Office 临时文件默认不提交到 Git，避免上传大文件和敏感业务资料。

## 主要代码模块

- `code/main.py`：程序入口。
- `code/runner.py`：主要填报流程调度。
- `code/registry.py`：数据源和报表页签配置。
- `code/intransit_read.py`：读取原油暂估表，在途数量和人民币金额。
- `code/intransit_price_pack.py`：读取在途美元/桶价格包，用于 2-3 等报表逻辑。
- `code/intransit_write.py`：将在途明细写入成本报表。
- `code/sheet_2_3_fill.py`：2-3 表相关填充逻辑。
- `code/ui_app.py`：界面程序相关逻辑。

## 当前重点问题

历史数据测试基本正常，但新增月份经常出现部分数据取不到。当前重点关注：

- 2026 年 5 月“其他在途”未取数问题。
- 暂估表页签名、列名、标题行、合并单元格、隐藏列等格式变化。
